# Event Catalog

## Purpose

This document defines the event types produced by the Microsoft Fabric Smart Farming Analytics Platform.

The Event Catalog serves as the authoritative reference for all streaming events generated by the platform. It describes the purpose of each event, the producing system, the primary consumers, and the expected event frequency.

All events conform to the Canonical Event Contract defined for this project.

---

# Canonical Event Contract

Every event generated by the platform shall include the following metadata fields.

| Field | Description |
|--------|-------------|
| event_id | Globally unique identifier for the event |
| event_type | Identifies the business event |
| event_timestamp | UTC timestamp when the event occurred |
| ingestion_timestamp | UTC timestamp assigned during ingestion into Microsoft Fabric |
| schema_version | Version of the event schema |
| facility_id | Unique identifier for the farming facility |
| zone_id | Climate-controlled growing zone |
| correlation_id | Identifier used to correlate related events |
| producer_id | Source application or device generating the event |
| environment | Deployment environment (dev, test, prod) |
| payload | Business-specific event data |

---

# Event Categories

The platform generates six primary categories of events.

| Category | Description |
|----------|-------------|
| Environmental Telemetry | Environmental sensor readings |
| Hardware Metrics | Equipment health and performance |
| Crop Batch Events | Crop lifecycle changes |
| Maintenance Events | Maintenance activities |
| System Events | Platform operational events |
| Alert Events | Critical operational alerts |

# Event Definitions

## EVT-001: Sensor Telemetry Event

### Event Type

sensor.telemetry

### Description

Represents a real-time environmental sensor reading captured from a growing zone.

### Producer

Python Smart Farm Simulator

### Consumers

- Eventstream
- Eventhouse
- Data Activator
- OneLake Lakehouse

### Frequency

Every 5 to 15 seconds per sensor.

---

## EVT-002: Hardware Metrics Event

### Event Type

hardware.metrics

### Description

Reports the operational status and performance metrics of farming equipment.

### Producer

Python Smart Farm Simulator

### Consumers

- Eventstream
- Eventhouse
- Data Activator
- OneLake Lakehouse

### Frequency

Every 10 to 30 seconds per monitored device.

---

## EVT-003: Crop Batch Event

### Event Type

crop.batch.lifecycle

### Description

Represents lifecycle events for crop batches, including planting, stage transitions, and harvest completion.

### Producer

Python Smart Farm Simulator

### Consumers

- Eventstream
- Eventhouse
- OneLake Lakehouse

### Frequency

Generated when lifecycle changes occur.

---

## EVT-004: Maintenance Event

### Event Type

maintenance.activity

### Description

Captures planned and unplanned maintenance performed on farming equipment.

### Producer

Python Smart Farm Simulator

### Consumers

- Eventstream
- Eventhouse
- OneLake Lakehouse

### Frequency

Generated on maintenance activities.

---

## EVT-005: Platform Event

### Event Type

platform.system

### Description

Represents platform operational events such as simulator startup, shutdown, or processing errors.

### Producer

Python Smart Farm Simulator

### Consumers

- Eventstream
- Eventhouse
- Platform Monitoring Dashboard

### Frequency

Generated when platform events occur.

---

## EVT-006: Alert Event

### Event Type

alert.critical

### Description

Represents a critical environmental or equipment condition requiring immediate attention.

### Producer

Microsoft Fabric Data Activator
(Eventhouse rule evaluation)

### Consumers

- Real-Time Operations Dashboard
- Farm Operators
- Operations Managers
- Microsoft Teams
- Email Notifications

### Frequency

Event-driven.

# Event Versioning

All events include the `schema_version` field.

Schema evolution shall follow these principles:

- New optional fields may be added without breaking compatibility.
- Existing fields shall not be renamed.
- Existing fields shall not change data types.
- Breaking schema changes require a new schema version.

---

# Event Naming Standards

Event types follow the convention:

```

<domain>.<entity>[.<action>]

```

Examples:

- sensor.telemetry
- hardware.metrics
- crop.batch.lifecycle
- maintenance.activity
- platform.system
- alert.critical

# Design Considerations

The Event Catalog is the foundation for:

- Eventstream routing
- Eventhouse ingestion
- Data Activator rule evaluation
- OneLake Lakehouse persistence
- Spark Notebook transformations
- Fabric Data Factory pipelines
- Fabric Warehouse loading
- Power BI semantic models

Any new event type introduced during future development must first be documented in this catalog before implementation.

# Architecture Alignment

The Smart Farming Analytics Platform processes every event using the following architecture:

1. Python Smart Farm Simulator publishes events.
2. Eventstream ingests and routes streaming telemetry.
3. Eventhouse stores streaming data for operational analytics.
4. Data Activator evaluates streaming events for alert conditions.
5. Eventhouse continuously persists telemetry into the OneLake Lakehouse Bronze layer.
6. Spark Notebooks transform historical data through the Bronze, Silver, and Gold layers.
7. Fabric Data Factory loads Gold datasets into the Fabric Warehouse.
8. Power BI consumes curated historical datasets for analytical reporting.
